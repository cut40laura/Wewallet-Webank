# 验证码和账号代操作

案例类型：账户控制权异常

公开来源线索：
涉贷中介和伪冒诈骗常诱导客户添加私人联系方式、下载非官方链接或交出账号验证码，进而代操作贷款申请。

适用场景：客户申请流程由第三方远程指导或代办，客户本人不清楚页面授权和提交内容。

常见表现：
- 客户把验证码、身份证、银行卡、营业执照发给中介。
- 第三方远程控制手机或电脑，代为点击授权。
- 客户不知道征信授权、企业授权、贷款合同内容。
- 登录设备、IP、手机号或收款账户与客户常用信息不一致。
- 客户回答问题前等待中介提示。

客户经理内部判断：
验证码和账号代表身份、授权和资金控制权。代操作会带来冒名申请、信息泄露、合同意愿不真实和资金被转走风险。

材料核验：
- 申请设备、手机号、登录记录。
- 征信和企业授权确认。
- 客户本人对申请金额、用途、还款责任的理解。
- 是否存在远程控制软件、私人链接或陌生账号绑定。

温和追问：
1. 申请过程中是否有第三方帮您操作手机或电脑？
2. 验证码和登录账号是否一直由您本人保管？
3. 贷款金额、用途、合同责任是否由您本人确认？
4. 是否点击过非官方链接或安装过远程控制软件？

客户可见边界：
可以说“验证码不能给任何人，贷款申请和授权必须由本人在官方渠道完成”。不要指导客户继续让中介代办。
