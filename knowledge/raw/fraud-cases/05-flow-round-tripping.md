# 流水循环做大规模

案例类型：资金流水异常

适用场景：客户提供的流水显示交易频繁、金额较大，但难以证明真实经营交易。

常见表现：
- 多笔资金在同日或短期内进出，余额停留时间短。
- 交易对手少且重复，备注为“货款”“往来款”“备用金”等笼统描述。
- 对公账户和个人账户之间频繁互转。
- 资金流入后快速转回原付款方或关联账户。
- 流水规模高，但毛利、纳税、库存、员工和场地不支撑。

客户经理内部判断：
流水高不等于经营强。循环流水常用于制造交易活跃假象，应核验交易对手、业务合同、发票、物流和资金最终用途。对公和个人混用时需区分真实经营收付款和个人周转。

材料核验：
- 交易对手清单和业务关系说明。
- 对应合同、订单、发票、物流或验收记录。
- 大额往来款的形成原因和还款安排。
- 关联企业或个人账户关系说明。

温和追问：
1. 这些大额流水分别对应哪些订单或客户？
2. 同一交易对手短期多次进出款的业务背景是什么？
3. 对公账户和个人账户之间互转主要用于什么经营环节？
4. 是否能按几笔大额交易提供合同、发票和收货记录？

客户可见边界：
不要说“刷流水”。可以说“流水需要和真实交易资料匹配，我们选几笔大额交易补充说明即可”。
