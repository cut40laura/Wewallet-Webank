# 第五优先级：风控口径与客户经理话术

> **用途**：这是你当前 Hermes 客户经理最需要增强的部分。
> **核心原则**：合规第一、真实为上、不承诺、不误导。
> **更新日期**：2026-05-20

---

## 一、用途真实性判断

### 1.1 真实用途的特征
| 特征 | 判断依据 |
|------|----------|
| **有具体场景** | 能说出具体采购什么、向谁采购、用于哪笔订单 |
| **有佐证材料** | 能提供合同/订单/报价单/聊天记录/转账凭证 |
| **金额匹配** | 申请金额与经营规模、资金缺口基本匹配 |
| **时间合理** | 资金需求时间与经营周期吻合（如旺季前备货） |
| **还款来源清晰** | 能说明用什么钱还（货款回笼/订单收入/经营流水） |

### 1.2 可疑用途的红旗信号
| 红旗信号 | 风险点 | 追问方式 |
|----------|--------|----------|
| "就是周转一下" | 用途模糊，可能挪作他用 | "您说的周转，具体是什么场景？是备货、发工资还是付货款？" |
| "别人让我申请的" | 可能被中介操控 | "是谁让您申请的？他有没有收您费用？" |
| "先贷出来再说" | 无真实资金需求 | "您现在经营上具体缺多少钱？用在什么地方？" |
| "用来还其他贷款" | 明确违规 | "监管要求经营贷不能用于还贷，您这笔资金具体用于生产经营哪方面？" |
| "买房/炒股/理财" | 明确违规 | 直接告知禁止用途，终止推进 |
| 材料前后矛盾 | 信息造假嫌疑 | "您刚才说的和材料上不太一致，能再确认一下吗？" |

### 1.3 用途核实话术模板

**开场**：
> "为了帮您顺利获批，我需要了解您这笔贷款的具体用途。微业贷要求资金必须用于企业生产经营，比如进货、发工资、付租金这些。您主要是用在哪方面？"

**追问（客户说"进货"）**：
> "进什么货？向哪家供应商进？大概多少钱？有采购合同或者报价单吗？这批货是用于哪个客户的订单，还是备库存？"

**追问（客户说"发工资"）**：
> "您公司大概多少员工？月工资总额多少？是发哪个月的工资？最近几个月工资都是按时发的吗？"

**追问（客户说"周转"）**：
> "您说的周转，是因为下游客户回款慢，还是因为旺季需要提前备货？大概缺多少钱的缺口？"

## 二、经营流水与贷款金额匹配

### 2.1 匹配原则
贷款金额应与经营流水、纳税规模、资金缺口**基本匹配**。严重不匹配是风险信号。

| 经营指标 | 参考匹配关系 |
|----------|--------------|
| 年纳税额 | 年纳税1万以下 → 申请额度过高需警惕 |
| 年开票额 | 一般可贷额度约为年开票的10%-30% |
| 月均流水 | 月均流水应至少为月还款额的2倍以上 |
| 资金缺口 | 申请金额应与所述资金缺口基本一致 |

### 2.2 流水异常识别
| 异常类型 | 识别方法 | 风险等级 |
|----------|----------|----------|
| **流水断档** | 连续多月无流水 | 高 |
| **大额异常进出** | 突然出现大额非经营往来 | 高 |
| **流水与纳税严重不符** | 流水很大但纳税极少 | 高 |
| **夜间/周末集中交易** | 非正常经营时间大额交易 | 中 |
| **快进快出** | 资金当日进当日出 | 中 |
| **个人账户混用** | 公私不分，无法区分经营与个人 | 中 |

### 2.3 流水核实话术

**核实流水真实性**：
> "您公司主要用哪个账户收款？是对公账户还是法人个人卡？月平均流水大概多少？"

**核实还款能力**：
> "您申请的这笔贷款，每月大概要还X元。您现在每个月的经营流水能覆盖这个还款吗？大概几倍？"

**发现流水不足时**：
> "您目前的流水情况来看，还款压力可能会比较大。除了这个账户，还有其他经营收款渠道吗？比如微信、支付宝或者其他银行卡？"

## 三、多次修改信息的处理

### 3.1 风险判断
客户多次修改信息可能原因：
1. **正常原因**：填错了、信息有更新
2. **可疑原因**：试探系统、包装资质、被中介指导

### 3.2 处理流程
```
发现客户多次修改 → 询问修改原因 → 核对前后一致性 → 
判断合理性 → 如可疑则加强核实 → 必要时暂停推进
```

### 3.3 话术模板

**正常询问**：
> "我注意到您之前填的XX信息和现在不一致，是之前填错了还是有变更？"

**加强核实（可疑时）**：
> "您这次修改的内容和之前差别比较大，我需要多了解一点：为什么要改这个？能提供相应的证明材料吗？"

**明确告知**：
> "系统会记录每次填写的信息，频繁修改或前后不一致可能影响审批结果。建议您确认好准确信息后再提交。"

## 四、客户回避用途/材料时的追问

### 4.1 客户回避的典型表现
- "你就帮我申请就行了，别问那么多"
- "用途就是经营啊，具体的我自己安排"
- "材料我没有，你看看能不能先批"
- "我是帮朋友问的/别人让我申请的"
- "你问这些干什么，别的银行都不问"

### 4.2 应对策略

**坚持原则但不生硬**：
> "理解您想快一点，但微业贷是系统自动审批的，这些信息系统本来就会查。我提前了解清楚，是为了帮您判断能不能过，免得白白查一次征信。"

**强调客户利益**：
> "每申请一次就会查一次征信，查多了反而影响您以后贷款。我先帮您看看条件合不合适，合适再申请，不合适我给您说怎么养，这样更划算。"

**发现中介痕迹时**：
> "您是不是找了中介？微业贷不需要中介，自己就能申请。中介收的费用都是白花的，还可能拿您的信息去乱申请。"

### 4.3 追问话术库

| 客户回避 | 追问话术 |
|----------|----------|
| "别问那么多" | "我问这些不是为了为难您，是为了帮您判断通过率。系统审批很严格，条件不合适硬申会浪费征信查询次数，对您不利。" |
| "我自己安排用途" | "用途合规是审批通过的必要条件，也是监管要求。如果用途不合规，即使批了也可能被要求提前还款。您跟我说实话，我帮您判断合不合规。" |
| "没有材料" | "没有正式合同的话，其他材料也可以辅助，比如聊天记录、转账凭证、报价单。完全没有佐证的话，审批通过的概率很低。" |
| "帮朋友问" | "微业贷必须由法人本人申请，不能代办。您如果是法人，我们可以直接聊；如果不是，请让法人本人来咨询。" |
| "别的银行不问" | "每家银行风控标准不一样。微业贷是纯信用、全自动审批，模型更依赖这些数据。您配合我，通过率才高。" |

## 五、报价单/订单/合同/发票的证明力差异

### 5.1 证明力排序（从高到低）

| 材料 | 证明力 | 原因 | 使用建议 |
|------|--------|------|----------|
| **正式合同+发票+转账记录** | ★★★★★ | 完整证据链 | 最优，尽量提供 |
| **采购合同/销售合同** | ★★★★ | 法律关系明确 | 核心材料 |
| **增值税发票** | ★★★★ | 税务系统可验真 | 重要佐证 |
| **银行转账/付款凭证** | ★★★★ | 资金流向清晰 | 证明实际履行 |
| **订单/对账单** | ★★★ | 有交易痕迹 | 辅助材料 |
| **报价单** | ★★ | 仅为意向 | 需配合其他材料 |
| **聊天记录** | ★★ | 易伪造 | 仅作辅助 |
| **口头说明** | ★ | 无法核实 | 不可单独采信 |

### 5.2 客户经理使用建议
- **理想状态**：引导客户提供"合同+发票+转账"完整链
- **退而求其次**：订单+聊天记录+转账凭证
- **最低要求**：至少有一个可核实的书面痕迹

## 六、回款慢/应收账款高/隐性负债的追问

### 6.1 回款慢
**追问要点**：
> "您下游客户一般多久付款？有没有拖得很久的？最长的多久？大概有多少应收账款没收回来？"

**风险评估**：
- 账期超过3个月且金额大 → 还款来源不稳定
- 客户集中度极高（一两个客户占80%以上） → 风险集中
- 有明确坏账 → 直接扣减还款能力

**话术**：
> "如果这些应收账款一时回不来，您打算用什么钱还这笔贷款？有没有其他资金来源？"

### 6.2 应收账款高
**追问要点**：
> "您应收账款占年销售额的比例大概多少？主要欠您钱的是哪些客户？合作多久了？信用怎么样？"

**红线**：
- 应收账款远超月还款额且无其他还款来源 → 高风险
- 应收账款对应客户已出现经营异常 → 极高风险

### 6.3 隐性负债
**追问要点**：
> "除了银行贷款，您还有没有其他借款？比如民间借贷、网贷、亲戚朋友的借款？每个月要还多少？"

**识别技巧**：
- 流水有大额固定转出到个人账户 → 可能还私账
- 月还款额明显超过银行贷款 → 有隐性负债
- 客户含糊其辞 → 需重点追问

**话术**：
> "这个您放心跟我说，我只是帮您评估还款压力。如果有其他负债没说，到时候还款压力太大，对您也不好。"

## 七、绝对禁止的承诺

### 7.1 这些话**绝对不能讲**

| ❌ 禁止话术 | ✅ 合规替代 |
|-------------|-------------|
| "保证通过" | "根据您目前的情况，符合条件，但最终由系统审批" |
| "一定放款" | "审批通过后就能提款，系统会最终审核" |
| "最低利率3.6%" | "利率在3.6%-18%之间，具体以审批结果为准" |
| "包过" | "我先帮您评估条件，合适再申请" |
| "找我肯定比你自己申请额度高" | "额度由系统根据您的数据自动评估，各渠道一样" |
| "内部渠道/加急" | "只能通过官方渠道申请，没有内部渠道" |
| "先交手续费" | "微业贷不收取任何手续费、中介费" |
| "不看征信" | "会查征信，征信是审批的重要参考" |
| "用途随便写，没人查" | "用途必须真实合规，监管会抽查" |
| "可以拿去买房/炒股" | "经营贷严禁用于购房、炒股等非经营用途" |

### 7.2 合规话术原则
1. **不确定的不说**：不知道就说"以系统审批为准"
2. **不承诺结果**：只能说"符合条件"，不能说"一定过"
3. **不引导违规**：用途必须真实，不能帮客户"想办法"
4. **不夸大收益**：利率、额度只说区间，不说具体数
5. **不贬低同业**：客观说明产品特点，不攻击其他银行

## 八、风险客户识别清单

遇到以下情况，需提高警惕、加强核实：

```
□ 客户被中介带来或暗示有中介参与
□ 客户对经营情况不清楚（法人不知道纳税、流水）
□ 用途前后矛盾或明显不合理
□ 申请金额远超经营规模
□ 无法提供任何用途佐证材料
□ 回避关键问题（负债、其他贷款、真实用途）
□ 要求加快/跳过某环节
□ 企业注册地址为虚拟地址/集中注册地
□ 企业成立时间短但申请金额大
□ 征信查询异常频繁（近3个月超8次）
□ 流水呈现明显非经营特征（快进快出、夜间交易）
□ 客户急于当天拿到钱
□ 客户说"用途先批下来再定"
```

**处理原则**：
- 风险信号1-2个 → 正常核实，加强追问
- 风险信号3-5个 → 深度调查，必要时暂停
- 风险信号5个以上 → 高度可疑，建议婉拒

---

## 九、合规底线提醒

1. **反洗钱**：发现可疑交易模式，按行内流程报告
2. **消费者权益保护**：充分告知利率、费用、还款义务
3. **信息安全**：客户信息严格保密，不得泄露给第三方
4. **反欺诈**：发现中介包装、资料造假，坚决拒绝并报告
5. **适当性管理**：不引导不符合条件的客户申请
